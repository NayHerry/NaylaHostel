Perfect — you’ve already got your backend and frontend folders pushed into a single GitHub repo. Let’s walk through the **step‑by‑step deployment process on Render** for your Django project with PostgreSQL, automatic superuser creation, migrations, and static files collection.

---

## 1. Prepare Your Codebase
- **requirements.txt**: Make sure it includes:
  ```txt
  django
  gunicorn
  dj-database-url
  psycopg2-binary
  whitenoise
  ```
- **settings.py** (backend/core/settings.py):
  - Add `dj_database_url` to configure PostgreSQL:
    ```python
    import dj_database_url

    DATABASES = {
        'default': dj_database_url.config(
            default=os.environ.get('DATABASE_URL')
        )
    }
    ```
  - Add Whitenoise for static files:
    ```python
    MIDDLEWARE = [
        'whitenoise.middleware.WhiteNoiseMiddleware',
        # ... other middleware
    ]

    STATIC_URL = '/static/'
    STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')
    STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'
    ```

---

## 2. Push to GitHub
- You’ve already pushed both `backend` and `frontend` directories.
- Ensure `.gitignore` excludes `venv/` and `db.sqlite3`.

---

## 3. Create a Render Web Service
- Go to [Render](https://render.com).
- Create a **new Web Service** → connect your GitHub repo.
- Select **backend** as the root directory (Render lets you set the working directory).
- Choose **Python 3.11+** environment.

---

## 4. Configure Environment Variables
In Render dashboard → **Environment** tab:
- `DATABASE_URL` → Render will provide PostgreSQL connection string when you add a PostgreSQL service.
- `SECRET_KEY` → generate a Django secret key.
- `DEBUG` → set to `False`.

---

## 5. Add PostgreSQL Database
- In Render → create a **PostgreSQL service**.
- Copy its connection string → paste into `DATABASE_URL` env variable of your web service.

---

## 6. Build & Start Commands
In Render service settings:
- **Build Command**:
  ```bash
  pip install -r backend/requirements.txt
  python backend/manage.py migrate
  python backend/manage.py collectstatic --noinput
  ```
- **Start Command**:
  ```bash
  gunicorn core.wsgi:application --chdir backend
  ```

---

## 7. Auto‑Create Superuser
Add a script in `backend/core/management/commands/create_superuser.py`:

```python
from django.core.management.base import BaseCommand
from django.contrib.auth import get_user_model
import os

class Command(BaseCommand):
    def handle(self, *args, **kwargs):
        User = get_user_model()
        username = os.environ.get("DJANGO_SUPERUSER_USERNAME", "nayla")
        password = os.environ.get("DJANGO_SUPERUSER_PASSWORD", "1234")
        email = os.environ.get("DJANGO_SUPERUSER_EMAIL", "nayla@example.com")

        if not User.objects.filter(username=username).exists():
            User.objects.create_superuser(username=username, email=email, password=password)
            self.stdout.write(self.style.SUCCESS("Superuser created"))
```

Then update **Build Command** in Render:
```bash
pip install -r backend/requirements.txt
python backend/manage.py migrate
python backend/manage.py create_superuser
python backend/manage.py collectstatic --noinput
```

---

## 8. Deploy & Verify
- Render will run build commands, apply migrations, create superuser `nayla/1234`, and collect static files.
- Visit your Render URL → login at `/admin` with `nayla / 1234`.
